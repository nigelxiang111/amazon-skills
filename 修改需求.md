# 缺失MCP:
- **缺失卖家精灵MCP**：卖家精灵是亚马逊卖家常用的工具，缺失该MCP可能会影响对市场数据的分析和决策制定。应该由卖家精灵和sorftime共同合作获取数据。卖家精灵的数据优于sorftime，注意数据不要获取重复，减少MCP的调用次数，需要缓存命中。
# 修改需求：
- **增加卖家精灵MCP**：需要增加卖家精灵MCP，并按照卖家精灵MCP的工具，调整SKILLS的调用方式，确保能够正确获取卖家精灵的数据。
- **修改数据获取逻辑**：需要修改数据获取逻辑，优先从卖家精灵获取数据，如果卖家精灵没有数据，再从sorftime获取数据。
- **增加数据缓存机制**：需要增加数据缓存机制，减少MCP的调用次数，提高系统性能。
- **softime补充机制**：1.如果卖家精灵没有数据，softime需要补充数据，确保系统能够正常运行。2.sorftime存在重要的数据且卖家精灵没有数据的情况下，softime需要补充数据，确保系统能够正常运行。
- **数据去重机制**：需要增加数据去重机制，确保从卖家精灵和sorftime获取的数据不会重复，避免数据冗余和混乱。
- **数据保存机制**：需要增加数据保存机制，将获取的数据通过python的pandas或其它方式进行处理后，保存到本地excel文件中，excel可以建立多个sheet，存放不同类型的数据，方便后续分析和使用，并注明数据来源和获取时间。excel文件命名规范为"数据类型_获取日期时间.xlsx"，例如"市场分析_2024-06-30_112032.xlsx"。
- **jina-mcp-server** : jina MCP可能存在连接不上或权限没有的问题，可以换成fetch方式获取数据，都不可以后可以跳过。
